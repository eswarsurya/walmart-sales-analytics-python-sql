# Walmart Sales Analytics Python SQL

Retail analytics project using Python, SQL-style analysis, and business reporting to explore Walmart weekly sales performance, store-level trends, holiday sales behaviour, and external business factors.

## Why I Built This

Retail data is a strong way to practise end-to-end analytics because it connects data cleaning, sales trends, store performance, external drivers, and business recommendations. This project shows how a raw sales dataset can be converted into a clear business review.

## Business Questions

- Which stores generated the highest total weekly sales?
- How did sales change across 2010, 2011, and 2012?
- Do holiday weeks perform differently from non-holiday weeks?
- How do fields such as temperature, fuel price, CPI, and unemployment relate to weekly sales?
- How can Python and SQL-style summaries support retail performance reporting?

## Tools Used

Python, Pandas, SQL, Jupyter Notebook, Excel, PowerPoint, and business analytics reporting.

## What I Implemented

I prepared and explored store-level weekly sales data, created Python-based summary outputs, wrote SQL-style business analysis queries, and built public-safe evidence for GitHub review. I also checked the local project folder and avoided uploading unrelated SQL content or large files with user/product identifiers.

## Outputs And Results

The analysis shows Store 20, Store 4, Store 14, Store 13, and Store 2 as leading stores by total weekly sales. 2011 has the highest total sales in the dataset, while 2012 is a partial year ending in October. Holiday weeks show higher average weekly sales than non-holiday weeks, which is a useful retail planning signal.

## Project Walkthrough

A good way to explore this project is to start with `outputs/analysis_output_summary.md`, then review the sample data in `data/`, the SQL queries in `sql/`, and the Python workflow in `notebooks/`. The project is designed to show practical retail analytics using both Python and SQL-style thinking.
